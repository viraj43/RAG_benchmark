For years, Transformers have dominated modern AI. Models like GPT, Claude, and LLaMA are all built on attention mechanisms that process entire sequences at once. This approach enabled breakthroughs in language understanding, reasoning, and generation.

But a new architecture is gaining serious momentum: Mamba.

This is not just another model. It represents a fundamentally different way of thinking about sequence modeling, memory, and efficiency.

## The Core Difference

At a high level:

- Transformer uses attention over all tokens
- Mamba uses a compressed internal state

A Transformer processes each token by comparing it to every other token in the sequence. This creates a quadratic complexity problem:

O(n²)

As sequence length grows, computation and memory usage increase rapidly.

Mamba takes a different approach. Instead of revisiting all previous tokens, it maintains a dynamic internal state that evolves over time:

O(n)

This means it scales linearly, making it significantly more efficient for long sequences.

## How Transformers Work

Transformers rely on self-attention:

- Every token attends to every other token
- The model builds a full contextual map of the sequence
- Memory is stored explicitly via KV cache during inference

Advantages:

- Strong reasoning and in-context learning
- Precise retrieval of past tokens
- High accuracy across many tasks

Limitations:

- Expensive for long sequences
- Large memory footprint (KV cache grows with context)
- Slower inference at scale

## How Mamba Works

Mamba is based on State Space Models (SSMs). Instead of attention, it uses a recurrent state that updates as new tokens arrive.

Key idea:

The model does not remember everything. It selectively compresses information into a hidden state.

Advantages:

- Linear complexity
- Constant memory usage (no KV cache)
- Extremely fast inference
- Strong performance on long sequences

Limitations:

- Weaker explicit retrieval
- Less effective in few-shot prompting
- Reasoning still behind top Transformer models

## The Key Innovation: Selective State Space

Earlier SSMs struggled because they treated all inputs equally.

Mamba introduced a selective mechanism:

- Important tokens are retained
- Irrelevant tokens are discarded

This gives Mamba a behavior similar to attention, but without the full computational cost.

## Mamba-2: Bridging Theory

Mamba-2 improved the mathematical foundation:

- Unified view between attention and state space models
- Better training stability
- Improved scalability

This version made Mamba more competitive with Transformers in practice.

## Mamba-3: Inference-First Design

Mamba-3 represents a major shift. Instead of optimizing for training, it is designed for real-world deployment.

## 1\. Stronger Recurrence

Mamba-3 introduces a more expressive recurrence mechanism (exponential-trapezoidal update), allowing the model to capture more complex patterns over time.

## 2\. Complex-Valued States

Instead of real-valued states, Mamba-3 uses complex numbers.

This improves:

- Memory representation
- Temporal dynamics
- Long-context tracking

## 3\. MIMO (Multi-Input Multi-Output)

Multiple state channels operate in parallel.

Result:

- Higher accuracy
- No increase in decoding latency

This is unusual. Typically, more compute leads to slower inference. Mamba-3 avoids this tradeoff by utilizing underused GPU resources.

## Why Mamba-3 Matters

The AI landscape is changing.

The bottleneck is no longer training. It is inference.

Modern systems require:

- Real-time responses
- Long context handling
- Continuous generation (agents, coding systems, simulations)

Mamba-3 directly targets these needs:

- Faster decoding
- Lower memory usage
- Better scalability

In benchmarks, even small Mamba-3 models can outperform comparable Transformer models in speed, especially as sequence length increases.

## The Tradeoff: Retrieval vs Compression

Transformers store information explicitly.

Mamba compresses information.

This leads to a key tradeoff:

- Transformers excel at exact retrieval
- Mamba excels at efficient representation

For tasks requiring precise recall, Transformers still have an edge.

## The Future: Hybrid Architectures

The most likely direction is not replacement, but combination.

Hybrid models can:

- Use attention for retrieval
- Use Mamba for efficient sequence modeling

This allows systems to balance:

- accuracy
- speed
- memory usage

Several research directions and early systems are already moving toward this hybrid approach.

## Where Mamba Shines

Mamba is particularly effective in:

- Long document processing
- Streaming data
- Audio and speech modeling
- Genomics and biological sequences
- Real-time AI systems
- Agent-based architectures

## Final Thoughts

Transformers changed AI by making context accessible.

Mamba changes AI by making context efficient.

Mamba-3 is not just an incremental improvement. It reflects a deeper shift toward inference-optimized models that can operate at scale in real-world systems.

The question is no longer whether Transformers will be replaced.

The real question is how long it will take before Mamba-like architectures become a standard component of every serious AI system.

Title: Mamba, A New Approach That May Outperform Transformers
An architectural innovation improves upon transformers — up to 2 billion parameters, at least.

**What’s new:** Albert Gu at Carnegie Mellon University and Tri Dao at Princeton University developed the [Mamba](https://arxiv.org/abs/2312.00752?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) architecture, a refinement of the earlier state space sequence architecture. A relatively small Mamba produced tokens five times faster and achieved better accuracy than a vanilla transformer of similar size while processing input up to a million tokens long.

**Structured State Space Sequence (S4) basics:** [S4s](https://arxiv.org/abs/2111.00396?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY), also known as structured SSMs, can be functionally similar to recurrent neural networks (RNNs): They can accept one token at time and produce a linear combination of the current token and an embedding that represents all previous tokens. Unlike RNNs and their extensions including LSTMs — but like transformers — they can also perform an equivalent computation in parallel during training. In addition, they are more computationally efficient than transformers. An S4’s computation and memory requirements rise linearly with input size, while a vanilla transformer’s rise quadratically — a heavy burden with long input sequences.

**Key insight:** S4s are more efficient than transformers but, while a transformer’s input length is limited only by processing and memory, an S4’s input length is limited by how well its hidden state can represent previously input tokens as new tokens arrive. A [gating mechanism](https://arxiv.org/abs/1612.08083?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) that lets the model process the most important parts of an input and ignore the rest can enable it to process longer inputs. One viable gate: Typically S4s apply the same mathematical function to all input tokens, whose parameters consist of four learned matrices. Changing the matrices for each input enables the model to learn which tokens or parts of tokens are least important and can be ignored (set to zero). This condenses the input, enabling the modified S4 to process very long input sequences.

**How it works:** Mamba is made up of blocks, each of which includes a modified S4 (which the authors call a selective SSM). The authors pretrained different instances on a variety of tasks including generating tokens from [The Pile](https://arxiv.org/abs/2101.00027?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) (a collection of text from the web) and predicting DNA base pairs in [HG38](https://www.nature.com/articles/s41592-021-01252-x?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) (a single human genome) in sequences up to 1 million tokens long.

- In each block, the authors replaced three of the S4’s four fixed matrices with learned linear functions of the input. That is, they replaced each of three learned matrices with a learned matrix multiplied by the input. (The authors hypothesized that modifying the fourth matrix would not help, so they didn’t change it.)
- The following layer multiplied the model’s output with a linear projection of the block’s input. This acted as a gate to filter out irrelevant parts of the embedding.

**Results:** Mamba achieved better speed and accuracy than transformers of similar size, including tasks that involved inputs of 1 million tokens.

- Running on an Nvidia A100 GPU with 80GB, a Mamba of 1.4 billion parameters produced 1,446 tokens per second, while a transformer of 1.3 billion parameters produced 344 tokens per second.
- In sizes from 130 million parameters to 2.8 billion parameters, Mamba outperformed the transformer [Pythia](https://arxiv.org/abs/2304.01373?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) and the S4 [H3](https://arxiv.org/abs/2212.14052?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) on many tasks. It was better at predicting the next word of The Pile, and it was better at question-answering tasks such as [WinoGrande](https://arxiv.org/abs/1907.10641?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) and [HellaSwag](https://arxiv.org/abs/1905.07830?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY). For instance, on WinoGrande, using models of roughly 2.8 billion parameters, Mamba achieved 63.5 percent accuracy, Pythia 59.7 percent accuracy, and H3 61.4 percent accuracy.
- After fine-tuning on Great Apes DNA Classification (classifying DNA segments up to 1 million tokens long as belonging to one of five species of great ape), using models of 1.4 million parameters, Mamba achieved 70 percent accuracy, while [Hyena DNA](https://arxiv.org/abs/2306.15794?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY) achieved 55 percent accuracy.

**Yes, but:** The authors tested model sizes much smaller than current state-of-the-art large language models.

**Why it matters:** Google’s transformer-based Gemini 1.5 Pro offers context lengths up to 1 million tokens, but methods for building such models aren’t yet widely known. Mamba provides an alternative architecture that can accommodate very long input sequences while processing them more efficiently. Whether it delivers compelling benefits over large transformers and variations that provide higher efficiency and larger context is a question for further research

**We're thinking:** Research on Mamba is gaining momentum. Other teams are probing the architecture in projects like [Motion Mamba](https://arxiv.org/abs/2403.07487?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY), [Vision Mamba](https://arxiv.org/abs/2401.09417?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY), [MoE-Mamba](https://arxiv.org/abs/2401.04081?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY), [MambaByte](https://arxiv.org/abs/2401.13660?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY), and [Jamba](https://arxiv.org/abs/2403.19887?utm_campaign=The%20Batch&utm_source=hs_email&utm_medium=email&_hsenc=p2ANqtz-8Kh954rkXmE4vgpKvro3Klpjhn7IuT-Y_eXIYtgVIq9PTzwa5zFWX7FZZqv1tuDEEsTDuY).

## Overview

The Transformer architecture, dominant in large-scale AI models since its introduction in 2017, faces scalability limits as sequence lengths grow. A key bottleneck is that self-attention computation scales quadratically with context length, which can make processing very long sequences computationally inefficient.

## What Mamba Proposes

Recent research introduces a new architecture called Mamba, based on a "selective state space model" (selective SSM). This design generalizes the earlier S4 architecture (Structured State Spaces for Sequence Modeling) by allowing the model to selectively attend to or ignore incoming inputs. The modification makes certain parameters functions of the input, a small change that the authors report yields significant benefits.

## Performance Claims

According to the authors, Mamba can match or outperform Transformer models on language modeling tasks. It is reported to scale linearly with context length and to handle sequences up to million-token lengths in practice, with up to 5x inference throughput improvements in some settings. The authors also report state-of-the-art results across multiple modalities including language, audio, and genomics. Their Mamba-3B model is reported to outperform Transformer models of comparable size and to be competitive with Transformer models roughly twice its size on language modeling benchmarks.

## Relation to Prior Work

S4 has previously demonstrated effectiveness on long-range dependency benchmarks such as Long Range Arena and Path-X. S4 and other state space model approaches relate to RNNs, CNNs, and classical state space models (SSM). The Mamba paper compares its approach against other SSM-based methods and efficient attention variants, including linear attention, H3, Hyena, RetNet, and RWKV, many of which are used as baselines in the study.

## Authors and Contributions

The paper is authored by Albert Gu (Assistant Professor, Machine Learning, Carnegie Mellon University) and Tri Dao (Chief Scientist at Together.AI and incoming Assistant Professor of Computer Science at Princeton University). Albert Gu is credited with the selective SSM generalization of S4. Tri Dao is known for FlashAttention and subsequent work like Flash Attention v2 and Flash-Decoding, which optimize attention computation and memory usage for long-context inference. The two authors have collaborated previously.
