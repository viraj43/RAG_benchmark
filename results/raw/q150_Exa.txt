# 4 advanced Machine Learning techniques
[...]
What are the typical ways to improve a Machine Learning model? Adding more features. Performing selection to remove redundant features. Optimizing hyper-parameters. Ensembling several models together. Yes, all of this may work, but there are many other approaches to improve the score. Maybe some of them are a little bit less known, and some are not applicable in all cases, but when applied at the right time they can bring significant improvement. Let’s go through some of them.

For this reason, it is a good practice to use pseudo-labeling at the latest stages of model development, when it is already improved by feature engineering, parameter optimization, and other basic techniques.

### Outliers removal by model predictions
[...]
Removing outliers from training data is a standard step in most Machine Learning pipelines. But here I’m not talking about this simple standard way, which

[...]
A more advanced way exists for dealing with outliers. This technique allows not only to remove outliers in terms of too big/small values of a single feature but deals with outlier feature interactions as well.
[...]
- First, the model is trained and tuned in a standard way, including feature selection, hyper-parameters tuning, and so on.
- Then the predictions are made in OOF (Out-Of-Folds) fashion for the training data rows.
- The error (or any other relevant metric for the specific problem) of the prediction is calculated for each row.
- The rows with the biggest errors are defined as outliers. The percentage of rows to remove can vary in each case.
- The model is re-trained on the training data with the identified outlier rows removed.
[...]
From my experience, in many tasks, this approach gives slight improvement. The size of improvement obviously depends on how many outliers, bad features, and incorrect target values the particular dataset has. In some cases, however, it might happen, that most of the rows detected as outliers are in fact valid. This happens mostly due to a too weak model – for this reason, I would advise applying this approach only at the latest stages of model tuning

the existing model is already improved as high as possible with standard techniques.

### Data augmentation
[...]
In some way, this technique is similar to pseudo-labeling. In a sense that it helps to get more training data. And more training data typically results in a better model. The main difference is regarding the exact way how we get more training data.
[...]
If in pseudo-labeling we used additional real data with self-assigned labels. But in some cases, we simply don’t have any additional data at all. In the case of augmentation, we use real labels, but the data itself is modified and transformed in different ways to create new data with similar characteristics as the original dataset.
[...]
There are various techniques for data augmentation for all kinds of data – numeric data, time-series, image, and text data. For example, images can be rotated, cropped or skewed in different ways. From the logical perspective, the picture on the image (and therefore, the original label, too) is not changed. But for the model. this new image looks different and helps to make the model more robust.
[...]
For text data, one approach can be translation. We can translate the original sentence to some other language and then translate it back. Through the translation process, the original sentence will become slightly different but the meaning should still be the same – hence the original label is preserved.
[...]
### Explaining the model predictions
[...]
The machine learning model is sometimes considered a black box. But in reality, it is not. There’s plenty of information we can gather from the model about exactly how it is making decisions in one or another case. The exact amount of information we can get depends on the type of the model. Some models, like tree-based models or a linear regression, are very easy to explain. But, for example, neural network models provide less interpretable information about what’s happening inside.
[...]
In any case, it helps a lot to try our best to understand, how exactly the model is using the features. Looking at feature importance graphs is one way of doing that. If we spot some strange feature on top of the feature importance graph, it’s worth trying to find out if there is indeed some signal in the feature or if this is just noise. In the simplest case, try to remove the feature and compare the validation score with and without the particular feature.
[...]
But the feature importance graph is not the only source of information about the usage of features. There are more possibilities to dig much deeper inside the model internals. LIME and SHAP are some of those (I’ll leave the technical details regarding them for your own investigation, if interested). The benefit of these approaches is not only to check which features are used by the model but also to check the direction of feature impact. It gives us the understanding that lower values of a particular feature result in lower or higher values of the target. This knowledge allows us to use domain knowledge to judge if such a feature interpretation makes sense.
[...]
Deep investigation of features and their impacts on the model is not an easy task and is not fast, so in most cases it won’t be beneficial to do this for every single feature. But at least, it is worth checking the top important features and understanding how the model is using them and does this makes any sense. Depending on the result we might want to decide on removing the feature, or in some cases even completely reviewing our validation approach – if we see that using the obviously wrong feature leads to better validation scores.
[...]
In machine learning, the models are never really finished. There is always a possibility so squeeze out some additional tiny improvement. The key moment here is to stop improving at the right time – so that the model is "good enough", but also developed in a reasonable time.

Title: Gradient boosting
Key Highlights:
Gradient boosting is a machine learning technique based on boosting in a functional space, where the target is pseudo-residuals instead of residuals as in traditional boosting. It gives a prediction model in the form of an ensemble of weak prediction models, i.e., models that make very few assumptions about the data, which are typically simple decision trees. When a decision tree is the weak learner, the resulting algorithm is called gradient-boosted trees; it usually outperforms random forest. As with other boosting methods, a gradient-boosted trees model is built in stages, but it generalizes the other methods by allowing optimization of an arbitrary differentiable loss function.
[...]
The idea of gradient boosting originated in the observation by Leo Breiman that boosting can be interpreted as an optimization algorithm on a suitable cost function. Explicit regression gradient boosting algorithms were subsequently developed, by Jerome H. Friedman, (in 1999 and later in 2001) simultaneously with the more general functional gradient boosting perspective of Llew Mason, Jonathan Baxter, Peter Bartlett and Marcus Frean. The latter two papers introduced the view of boosting algorithms as iterative functional gradient descent algorithms. That is, algorithms that optimize a cost function over function space by iteratively choosing a function (weak hypothesis) that points in the negative gradient direction. This functional gradient view of boosting has led to the development of boosting algorithms in many areas of machine learning and statistics beyond regression and classification.
